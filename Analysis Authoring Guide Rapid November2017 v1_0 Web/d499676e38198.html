This section provides details for the Core Notations relating to Multiplicities (Cardinalities).
Multiplicities are annotates as follows:
• 
As ‘0’ for none - OR,
• 
A ‘*’ (star symbol) for an undefined many (this can also represent 0) - OR,
• 
A specific integer (e.g. ‘1’ or ‘5’ etc) to determine the exact quantity.
• 
Using two full stops or dots between the two represented values indicates a range of values, expressed by a following number / star symbol (e.g. ‘0..1’ or ‘1..*’),
◦ 
The double full stops may be omitted where there is no optionality (e.g. just a ‘1’ says that there must always be one of an item and no more than one).
• 
Multiplicities are further defined in this guide for use on Class Multiplicities on Associations as well as Class Attribute Multiplicities.
