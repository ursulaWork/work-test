“An Actor is a participant in a process / function. Actors may represent either human or system roles that interact with other modelled entities. They are represented according to the specific core notations for each diagram type that utilises actor roles (e.g. Actor elements on Use Case Diagrams or as Swimlanes on Activity Diagrams)”
Example figures showing the various representation of Actor roles on UML diagrams are available within each of the definitions for diagram types that represent Actors.
The purpose of Actors within UML are to represent interactions between model elements and human or system roles.
UML Diagram and element types / configurations for which Actor roles are defined include:
UML Diagram Type
Represented as
Use Case Diagram
Actor element as ‘stick-person’
Sequence Diagram Models
Actor element as ‘stick-person’ element with a descending lifeline
Activity Diagram
Actor defined as Swimlanes
Class Diagram
Actor defined as a Class, e.g. to representing a human / system role (this usage does not utilise a different kind of element / configuration and therefore has no additional content to describe it)
Further content to assist analysts to identify and understand actor roles within models is also available (via that link). As one means of organising Actor-related elements within a model and to make them available for reuse, the Analysis Methodologies have defined an Actors Catalogue Package to keep them all in one location.
The construction of UML Actor Elements are defined by their Core Notations — these are defined in the sections dealing with each specific usage.
