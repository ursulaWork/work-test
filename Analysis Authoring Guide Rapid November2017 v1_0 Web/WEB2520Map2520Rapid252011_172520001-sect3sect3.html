“A Package is a model structural concept used to describe ‘folders’ (and sub-folders) that hold a collection of related entities. A package provides a "namespace" for the grouped elements. A package can contain other Packages to define a hierarchical organisation. A package may also be represented as an element on diagrams (optionally with the first level of its contained entities also represented), as a short-hand depiction of all of its contents.”
- Packages may be used by both UML and non-UML modelling approaches within analysis models.
Examples figures showing Packages in context are available within the definitions for each usage (there is therefore no image available here).
Packages may be used within models in any of the following ways:
Package Type / Usage
Structural Model Packages.
Catalogue Packages.
Package Elements on Diagrams and Package Diagrams.
Packages are constructed according to their Core Notations (this also covers how they are used for each of the above contexts).
