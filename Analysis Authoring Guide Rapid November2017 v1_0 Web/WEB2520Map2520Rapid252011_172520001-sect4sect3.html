Product Name
Analysis Scope & Objectives Definition Document — (Scoping Doc)
Description
The Analysis Scope & Objectives Definition Document (a.k.a. the ‘Scoping Document’) describes what the analysis is trying to achieve for a defined domain, and is the first formal Artefact produced by Analysis Projects. A completed and approved Scoping Document is an expected output for the project; it is further expected that this document will be maintained throughout the lifecycle of the project, such that it always reflects the 'current' scope of the project. The Analysis Scope & Objectives Definition Document states the business aims for a project, including setting the business context of the work; and from that it derives the detailed scope for the Analysis, as well as identifying which Artefacts are to be produced
Status
The development status of the Analysis Scope & Objectives Definition Document according to the Analysis Methodology is ‘Mandatory’; it is publishable as a formal Deliverable Product.
Purpose
The purposes of the Analysis Scope & Objectives Definition Document are to:
•  Set the context of the work in terms of the business objectives, extant domain arrangements, its match to NHS Digital objectives and timescales
•  State the business objectives and needs to be addressed by the Project
•  Define the boundaries of the analysis work to be performed
•  Contribute to project planning by identifying the products expected to be developed for the specific Project in-hand
•  Form part of the mechanism used to gain approval to start / approval to proceed or continue with analysis work
•  Identify the Constraints to be applied, as well as any Exceptions to be included, and any Assumptions that need to be identified
•  State the formal ownership for product(s), as well as the governance controls to be applied to the project
•  Describe how the Analysis work fits within the wider project
•  Outline any Risks / Issues still outstanding at the conclusion of a Project (whether the work is completed, unfinished or to be mothballed)
Audience & Use
Internal:
•  Analysis Management specifically: Securing approval to Start / proceed / continue with domain work; and to meet stated Purpose for this Artefact.
•  Project Management: Inform project planning.
•  Domain Experts: Determine business scope.
External: 
•  Suppliers: Planning development and for information.
•  Users: Required for understanding of subsequent artefacts.
Format & Content
The Scoping Document is a Data Products artefact defined by the Rapid Methodology. It starts its development during the Planning and Discovery Stage of Analysis Projects. It may be developed externally to the Model using the MS Word-based Analysis Scope & Objectives Definition Template, and can have a life outside of the Model as an independent Artefact (i.e. it can be held under its own ID on FileCM); however a copy of it MUST always be embedded into the Analysis Model Artefact (no example figure available)
A link to the Document must also be placed on the Navigation Page within either the Model Management & Documents or Key Project Stage areas — the choice of which one is at the Model Authors discretion
