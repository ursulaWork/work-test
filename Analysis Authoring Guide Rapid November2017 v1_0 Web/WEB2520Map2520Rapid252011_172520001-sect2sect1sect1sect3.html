Each analysis artefact falls into one of three statuses. The status for each artefact are outlined within its Product Description.
Where an artefact is produced, then: 
•  It is publishable as a formal deliverable.
•  It is subject to the Review and Approvals process.
•  It should be listed within the Analysis Scope and Objectives Definition Document, as well as in the Analysis Review Checklist.
Status Definitions
Status
Definition
Mandatory:
This means that a Project artefact MUST be created and that therefore that a project cannot be considered complete without it being present.
Required:
This means that the Model Author does not have to produce the artefact in question (conditional upon the Analysis Methodology being employed), however this artefact SHOULD be developed. 
If a ‘Required’ artefact is not going to be developed, then the reasons for not doing so (as well as references to any replacement artefact/s) should be stated in the Analysis Scope and Objectives Definition Document. 
Optional:
This product MAY be developed for a project if it assists or supports a fuller understanding of the project domain or information requirements. Its inclusion is at the discretion of the project team, along with the project customer. As an Agile project progresses there may be a need to reconsider developing products that were initially rejected.
