“An Association specifies the relationship that exists between two entities on a diagram; they are annotated as ‘connectors’ between the various entities. The type of Association determines the nature of the relationship that exists between the two entities.”
Example figures showing each Association type are available in the relevant sections describing each specific relationship (there is therefore no example figures here).
The purpose of Associations is to identify where a relationship exists between elements. The range of different types of association available to UML diagrams differs dependent according to the diagram type and the elements being used, for which there are defined localised usage and construction parameters.
Informal relationships (i.e. those with no processing parameters) are typically identified using ‘Trace’ associations.
- Not all ‘relationships’ between diagram entities on other diagram types are annotated as ‘associations’ (e.g. ‘Flows’ on Activity Diagrams etc.,); these are dealt with in the relevant sections for the diagram-type they relate to.
Associations may be used on the following Diagram types:
Diagram types that use Associations (links are to Core Notations)
UML Class Diagram
UML Use Case Diagram
Requirements Diagram (a Non-UML methodology-defined use of associations)
