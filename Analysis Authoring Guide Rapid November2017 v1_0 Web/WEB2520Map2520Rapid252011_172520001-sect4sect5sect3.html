Purpose
The purpose of the Requirements Diagram Artefact is to:
•  Provide a diagrammatic representation of a projects Requirements Elements, using unambiguous project / domain-related language.
•  Show the relationships between requirements as associations between its elements, as well as identifying relationships to any other closely related elements (e.g. Use Cases).
•  Support Analysis Model testing and implementation by providing a basis for demonstrating the fulfilment of requirements.
•  Provide readers who are not familiar with modelling techniques with a point of entry into analysis / design artefacts.
Audience & Use
Internal:
•  Project Team: to meet stated Purpose for this Artefact,
•  Test Team: to inform test cases.
Description
A ‘Requirements Diagram’ is a Requirements Artefact. A separate Requirements Diagram should be held within each of the Classifying Sub-Packages (e.g. ‘Functional’ and ‘Non-Functional’) of the Requirements Catalogue. The Diagram elaborates the individual Requirements Elements defined for each classification, as well as any relationships between them
Format & Content
The Requirements Diagram is a Requirements Artefact modelled according to its own Adopted Notations. It is developed using the currently approved CASE Toolset (see: Enterprise Architect [EA].
Construction Guidelines
The construction of the Requirements Diagram Artefact is defined by its Adopted Notations.
