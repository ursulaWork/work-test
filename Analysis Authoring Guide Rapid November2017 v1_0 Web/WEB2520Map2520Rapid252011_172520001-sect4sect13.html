Heading Details
Product Name/s
Information Analysis Model (IAM)
Description
The Information Analysis Model (IAM) Artefact is based on a UML Class Diagram; it enables a fuller exploration and refinement of the area(s) of interest (the information and business concepts as classes) derived from the BIM Artefact that are being taken forward for subsequent analysis.
Status
The status for the IAM Artefact is ‘Optional’.
Purpose
The purpose of an IAM is to:
•  Provide a refinement of the concepts captured in the BIM and an elaboration of the information requirements.
•  Allow the exploration of the domain information requirements against known constraining data architectures, that are of significant interest to the solution.
•  Provide an information model for the Area(s) of Interest. It provides the domain information context for subsequent analysis / development.
Audience & Use
Internal:
•  Project Team: to determine the business data needs within the area of interest; and to meet stated Purpose for this Artefact.
•  Developers: as an input into development work.
•  Test Team: to inform test cases.
•  Technical Architects: to inform technical design.
External:
•  Domain Experts: verify the business data.
•  Suppliers: use as input into development work.
Format & Content
The Information Analysis Model Artefact comprises a UML Class Diagram and associated Classes within the package structure. The IAM artefact represents Class elements, their Attributes and the Associations between the Classes.
The IAM is developed during the Discovery Stage of a Project.
It is held within the model structure as follows: in a package named Information Analysis Model in the Area of Interest Package.
This artefact is created using the currently approved CASE toolset (Enterprise Architect)
