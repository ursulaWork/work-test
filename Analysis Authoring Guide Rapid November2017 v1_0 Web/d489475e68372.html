Purpose
After p in entry
The purpose of a Sequence Diagram Artefact is show a structured representation of behavior as a series of sequential steps over time.
It is used to:
• 
Identify Interactions (e.g. representing transmissions of communications) within a system or business process.
• 
Identify interface behaviours and requirements.
• 
Identify user stories and requirements.
• 
Provide verification of Use Cases or User Stories.
• 
Provide verification of static data content (and attributes).
• 
Providing verification of behaviour associated with data (and attributes).
• 
Define functional / implementation roles for Object and Actor Elements.
• 
Support the identification of Expected functionality.

Description
After p in entry
A Sequence Diagram Artefact represents the sequencing of a group of related interactions between entities (e.g. a communication or a message transfer between Actor roles and systems or within systems). These are represented using interactions (a transmission) between Lifeline Objects.
After p in entry
Format & Content
After p in entry
The Sequence Diagram Artefact comprises a UML Sequence Diagram and associated Objects within the package structure. The Sequence Diagram Artefact represents object / actor elements, and the Interactions between them.
The Sequence Diagram Artefact is developed during the Discovery Stage of  a Project.
It is held within the model structure as follows: in a package named Sequence Diagram Artefact in the Area of Interest Package:
• 
Where the model contains more than one sequence diagram these should be held in a suitably name package under the package.
• 
Objects representing Actors and Business Application roles should be shared across the model’s sub-packages.
This artefact is created using the currently approved CASE toolset (Enterprise Architect).
