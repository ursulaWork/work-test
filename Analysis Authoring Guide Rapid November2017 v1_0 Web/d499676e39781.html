Actors

An Actor is a participant in a process / function. Actors may represent either human or system roles that interact with other modelled entities. They are represented according to the specific core notations for each diagram type that utilises actor roles (e.g. Actor elements on Use Case Diagrams or as Swimlanes on Activity Diagrams)
Example figures showing the various representation of Actor roles on UML diagrams are available within each of the definitions for diagram types that represent Actors.
The purpose of Actors within UML are to represent interactions between model elements and human or system roles.
