Description
After p in entry
User Stories are short, simple descriptions of a feature told from the perspective of the person who desires the new capability, usually a user or customer of the system.

Status
After p in entry
The development status of User Story elements is ‘Required’.

Purpose
After p in entry
The purpose of User Stories is to agree with the Product Owner the functionality needed so that the User Story can be added to the product backlog for development.

Format & Content
After p in entry
User Stories are individual elements in the analysis model written in the syntax:
As a [role]:
I MUST / SHOULD / MAYto: [action]
So that: [goal]
They are created during the Discovery Stage and developed iteratively during delivery, and need to be completed as part of planning a Sprint.
They are held within the following model structure package/s: In the User Story Catalogue package.
They are created using the currently approved CASE Toolset.
