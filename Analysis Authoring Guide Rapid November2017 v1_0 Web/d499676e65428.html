Purpose
After p in entry
The purpose of individual Use Case Diagram Artefacts is to represent a defined view of the behaviours of a collection of related actor and use case elements within a ‘system’. The Use Case Diagram Artefacts within the analysis model together provide an organised view of its system behaviours (these are also elaborated as an overview using the Use Case Model Diagram Artefact). Use Case Diagrams also provide a useful starting-point for the development of other analysis / design artefacts.

Description
After p in entry
A UML Use Case Diagram elaborates the Use Case and Actor Elements, as well as the relationships (as Associations) between them, for a defined business subset within a Use Case Model
After p in entry
Format & Content
After p in entry
Use Case Diagram Artefacts are developed within Analysis Models using the currently approved CASE Toolset. They start their development during the TBC stage of a project, and are held in individually named use case sub-packages (with the same name as the use case they elaborate), located within the model hierarchy (in ascending order): TBC.
