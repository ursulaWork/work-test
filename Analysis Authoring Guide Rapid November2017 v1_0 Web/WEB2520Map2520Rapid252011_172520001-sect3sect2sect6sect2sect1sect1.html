This section outlines the Core Notations for Datatypes:
•  Datatypes are applied to class / object attributes to define a specification that its values MUST conform to.
•  Datatype identifiers are shown after the attribute name, preceded by a colon ‘:’ -
e.g. AttributeName: Datatype.
•  Datatypes are a form of Constraint.
•  Where an additional constraint exists upon a datatype (e.g. the identification of a defined vocabulary) this should be held as a constraint upon the class / object element.
•  A ‘Null Value’ may be specified by a datatype where no value is required, as distinct from a zero value‘ — different ‘flavours’ of null value may also be used by datatypes (as determined by the adopted methodology).
