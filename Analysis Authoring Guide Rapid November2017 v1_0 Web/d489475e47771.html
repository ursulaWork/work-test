The Analysis Model Artefact is an elaborated Analysis Model that contains the data, information specifications, Artefacts as well as other supporting content for an Analysis Project. The Analysis Model is defined as an Artefact in its own right, having its own set of development parameters, rules and instructions. It is elaborated using a series of Elements, Diagrams and Packages, as well as any supporting Documents.

The Analysis Model Artefact as the Analysis Model is defined in its own right as a formal artefact (for review purposes) by the Data Products Analysis Methodologies; as such it should always adhere to the guidelines outlined in this section.
It can be created using any defined CASE Toolset (or manually); however the use of the currently approved Toolset (currently: Enterprise Architect [EA]) and making use of the Analysis Model Template is strongly advised.
The purpose of the Analysis Model Artefact is to provide an organising structure (and repository) to hold all of the Elements, Diagrams and Artefact, as well as any supporting documents drawn together during an Analysis Project. These convey its analysis, information and data specifications using a series of Diagrams (as Artefacts) to represent its defined ‘views’.
The development status of the ‘Analysis Model Artefact’ is Mandatory in all cases. It is developed iteratively throughout the Project’s lifecycle, and it is publishable as a Formal Deliverable Product.
The content of the Analysis Model Artefact is defined by the chosen Methodology selected at the beginning of the Project. Each of the approved Methodologies has its own defined Artefact Set, model packages and structure, as well as Navigation Page layout.
There are no formal Adopted Notations for the Analysis Model Artefact as a whole, but its constituent artefacts and their Adopted Notations are fully defined.
NOTE: - There is no Product Description section for this artefact.
Extra note should also be taken however of the content for the Navigation Page, and its Adopted Notations.
