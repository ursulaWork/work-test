Structural Model Packages
Introduction ● Modelling and Project Concepts ● Packages within Analysis Models oNext: Catalogue Packages
A Structural Package is a folder (and sub-folders) that holds a collection of related entities, brought together to Model an information specification. Packages define the organising approach for Domain Analysis Models, providing the structure that identifies its business subsets, Artefacts and individual Elements. By placing Packages within Packages a structural hierarchy is also implied.

  Packages used to organise elements within a model
The purpose of a Package that defines a model’s structure is to provide an organising mechanism within models to support the storage and easy navigation of its entities and diagrams. Packages may also contain sub-packages / structures to support more complex content organisation schemes, such as illustrating model hierarchies.
NOTE: - Packages / folders may be used to keep all the elements of the same type together in a single location within a model, as in a Catalogue Package, and may also be represented as Package Elements on diagrams.
The construction of UML Packages as model structures is defined within the generic Packages Core Notations section.
