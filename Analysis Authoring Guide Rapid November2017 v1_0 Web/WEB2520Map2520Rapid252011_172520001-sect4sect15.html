Product Name
State Machine Diagram Artefact (State Diagram)
Status
The status of the State Machine Diagram Artefact is ‘Optional’.
Purpose
The purpose of the State Machine Diagram Artefact is to represent the various States of an entity as well as any transitional activities that are triggered at the end of a State:
•  It is used to:
–  Identify Interactions (e.g. representing transmissions of communications) within a system or business process.
–  Identify and verify triggers and activities in use cases or user stories.
–  Identify requirements.
–  Provide verification of behaviours of classes.
–  Provide verification of classes(and attributes).
Audience & Use
Internal:
•  Business Analysts: analysis technique to aid verification of functionality and/or process activities  along with static data
•  Project Team: to meet stated Purpose for this Artefact.
•  Test Team: to inform test cases.
External:
•  Domain Experts: verify the business processes and requirements.
•  Suppliers: use as input into development work.
Description
The State Machine Diagram Artefact is a UML State Machine Diagram that represents the different states an entity may move through during its lifecycle, as well as the triggers, conditions and process actions as it moves from one state to another.
Format & Content
The State Machine Diagram Artefact comprises a UML State Machine Diagram and associated States within the package structure. The State Machine Diagram Artefact represents State elements, and the Transitions between them.
The State Machine Diagram Artefact is developed during the ‘Discovery’ Stage of a Project.
It is held within the model structure as follows: in a package named State Machine Diagram Artefact in the Area of Interest Package.
This artefact is created using the currently approved CASE toolset (Enterprise Architect).
