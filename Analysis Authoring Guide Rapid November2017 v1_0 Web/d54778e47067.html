Acceptance Criteria
Introduction oNext: Wireframe Models
Acceptance Criteria are the conditions that a software product must satisfy to be accepted by a user, customer, or in the case of system level functionality, the consuming system. Acceptance criteria are a set of statements, each with a clear pass / fail result.
Acceptance criteria can be applied to any artefact that is intended to be used by testers (e.g. Requirements, Use Cases, User Stories etc.).
Acceptance Criteria as Test Criteria
These acceptance criteria define, at a high level, the ‘test criteria’ which can confirm that for example a User Story is working as required. These are not intended to be the full test scripts, but will be used and expanded into appropriate test scenarios and test scripts for use during Sprints, as necessary.
