A UML Class Model is a collection of Classes, that together define the information / data model for a specified domain subset. A UML Class Model is elaborated using one or more UML Class Diagrams, representing its Class Elements and their relationships to each other as Associations

  A UML Class Model elaborated using a Class Diagram (as a BIM Artefact)
The purpose of a UML Class Model is to capture the logical information concepts for a domain, as well as the business rules that govern the relationship between those concepts. Class models are developed iteratively as the understanding of the domain matures.
A UML Class Model is maintained in a named model package, which consists of:
• 
Class Diagram(s) — providing visual representations / elaboration of the class model.
• 
Class Elements — to capture the individual information specifications:
◦ 
These can be further defined to include:
▪ 
Attributes — with Multiplicity / Cardinality, may be defined using Datatypes (these can be displayed within the element).
▪ 
Behaviours — Class Operations represent the behaviour or services that a Class supports.
• 
Class Diagrams are also further elaborated using:
◦ 
Associations — define the relationships between Classes (with Cardinality / Multiplicity).
The construction of a UML Class Diagram, is defined by its Core Notations.
A Class Model is elaborated as a UML Class Diagram, upon which it represents its Classes and the Associations between them.
