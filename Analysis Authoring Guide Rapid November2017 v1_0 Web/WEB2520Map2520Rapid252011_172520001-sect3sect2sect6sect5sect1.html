This section outlines the Core Notations for Actors. Actors are annotated differently according to which diagram type is being employed at the time:
•  Actors are typically represented on diagrams using a ‘stick-person’ element.
Where an Actor role is shown on a UML Class Diagram it is represented using a UML Class Element.
•  Where an Actor role is shown on a UML Activity Diagram it is represented using a swimlane.
•  A useful organising approach to managing the individual Actor elements and roles within a model is to keep them in a single location within a ‘Catalogue’ Package’. Locating them in a single package supports reuse and helps prevent duplication of roles (this is the preferred methodologies approach for the storage of Actor roles within its Models, e.g. Actors Catalogue Package Artefact).
•  Further information is also available (via the links) for identifying and understanding actors within models, and on specific diagram types.
