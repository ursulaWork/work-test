Product Name/s
Business Information Model Artefact — (BIM)
Description
The Business Information Model (BIM) Artefact is based on a UML Class Diagram; it provides a basic overview of the information concepts (as Classes) for a defined business area, as well as showing where these may have a relationship to each other (as Associations).
Status
The status of the BIM Artefact is ‘Required’.
Purpose
The purposes of the BIM are:
•  To illustrate the major information concepts within the project / domain and the associations between them.
•  Inform and advise the scope of a project, providing an appropriate frame of reference for subsequent analysis.
•  A general purpose tool for any discussion of the information content of the project / domain.
•  To identify the business area(s), information concepts and system(s) with which the project / domain is concerned.
•  For identification of information concepts (Classes) that are shared with other projects / domains.
Audience & Use
Internal:
•  Project Team: to inform and advise on the project scope; and meet stated purpose for this artefact
•  Domain experts: to agree overall information concepts of business area
External:
•  All: to provide an overview and introduction to the information concepts within the project / domain
Format & Content
The Business Information Model comprises a UML Class Diagram and associated Classes within the package structure. The BIM artefact represents Class elements and the Associations between them.
It is developed during the ‘Inception’ Stage of a Project.
It is held within the model structure as follows: in a package named Business Analysis Model in the ‘Rapid Analysis Methodology’ package.
This artefact is created using the currently approved CASE toolset (Enterprise Architect).
