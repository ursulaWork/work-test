Purpose
After p in entry
The purpose of a Use Case Model is to model the process behaviours and interactions for an Analysis Project (e.g. between Actors and Systems / processes). It also provides a single model repository to hold all of its Use Case related entities (packages, diagrams and elements) as well as all of the model’s Actors (These can also be held withinan Actors Catalogue Package, in the Use Case Model package).
NOTE: - The term ’Use Case’ is often used interchangeably to represent a complete modelled set of processes as well as to denote a single Use Case Element encompassing a single business process — within this guide the suffix ‘element’ defines it as a single modelled element.

Description
After p in entry
The Use Case Model Artefact is defined by its package, diagram and element contents (it is considered as an Artefact, for formal review purposes); together these elaborate different configurations of Use Case and Actor Elements
After p in entry
Format & Content
After p in entry
The Use Case Model Artefact comprises a UML Use Case Diagram and associated Use Cases within the package structure. The Use Case Diagram artefact represents Use Case and Actor elements and the Associations between them.
The development of the Use Case Model commences during the Discovery Stage of a project, and continues over the lifecycle of the project.
It is held within the model structure as follows: in a package named Use Case Model in the ‘Rapid Analysis Methodology’ package.
