“A wide-range of Relationships may exist between Entities within a Model. Relationships between Entities can be defined either as metadata ‘Links’, or modelled upon Diagrams, using different types of ‘Connectors’ (e.g. Associations, Flows etc.,). The type of Relationship defines how Entities may interact with or affect processing.”
The establishment of relationships between model entities may be defined by element metadata or toolset functionality (e.g. EA’s Relationship Matrix) — how these are achieved is determined by the CASE Toolset being used.
Connectors
Connectors
“‘Connectors’ are used to represent relationships between entities on a diagram (e.g. an association). The specific notation for a connector determines the way that the relationship may / should be understood or processed. The types of Connector used are determined by the diagram / model type being used. Typical examples of Connectors include: Associations, Interactions, Flows and Transitions.”
The main types of Connector utilised are those currently defined by the UML modelling approach (these can be used for both UML and non-UML modelling approaches), and are defined for each diagram-type, and may also be described in an entity’s metadata (CASE Toolset dependent).
The main types of Connectors used by diagrams include (there may also be diagram-specific variations on these):

Generic Connector types:
Trace Associations — these are for ‘Informal’ or ‘nominal’ relationships (i.e. those with no processing parameters) on diagrams between entities, these typically link to a non-processing entity (e.g. to hold additional explanatory content)
There are no generic Core Notations defined for UML Relationships — they are specifically defined for each application.
